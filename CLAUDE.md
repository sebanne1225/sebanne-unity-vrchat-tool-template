## Goal

- このツールで達成したいことを簡潔に書く。

## Current State

- 現在の実装状況、存在する画面、コマンド、主要ファイルを書く。

## Current Direction

- 現在進めている方向性、直近の優先事項を書く。

## Current Blocker

- 進行を止めている問題、未確定事項、依存関係を書く。

## Rules

- 非破壊を最優先にし、既存データや既存設定を直接書き換える前に確認手段を用意する。
- 冪等に動く実装を優先し、同じ操作を繰り返しても状態が壊れにくいようにする。
- Dry Run優先で、まずは変更予定の内容を確認できる導線を用意する。
- ログ重視で、処理対象、結果、スキップ理由、失敗理由を追跡できるようにする。
- 元データを直接壊さない。必要なら複製、退避、プレビューを使う。
- まず短い plan を出してから作業する。
- commit / push は明示的な指示があるまで行わない。
